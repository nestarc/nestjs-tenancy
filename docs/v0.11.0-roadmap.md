# v0.11.0 Roadmap

> 작성일: 2026-04-08 | 기준 버전: v0.10.1

## 리서치 배경

NestJS 멀티테넌시 생태계, Prisma 로드맵, 엔터프라이즈 SaaS 수요를 조사하여 v0.11.0 구현 내용을 도출.

---

## Tier 1 — 핵심 (v0.11.0 필수)

### 1. Tenant-Scoped Logger

**수요:** 모든 로그에 tenant ID를 수동으로 추가하는 보일러플레이트. 디버깅 시 테넌트별 필터링 불가.

```typescript
@Injectable()
export class OrderService {
  constructor(private readonly logger: TenantLogger) {}

  process() {
    this.logger.log('Processing order');
    // → [OrderService] [tenant:abc123] Processing order
  }
}
```

- NestJS `LoggerService` 래퍼, `AsyncLocalStorage`에서 자동 읽기
- 구조화 로그 지원 (`{ tenantId, message, ... }`)
- 기존 `TenancyContext`와 동일한 static storage 활용
- **난이도:** 낮 | **임팩트:** 중-높 | **예상 규모:** ~3 파일

### 2. Tenant-Aware Cache Interceptor

**수요:** 멀티테넌트 캐시 오염은 가장 빈번한 보안 버그. Redis/in-memory 캐시에서 tenant ID를 키에 포함하지 않으면 다른 테넌트 데이터가 반환됨.

```typescript
@UseInterceptors(TenantCacheInterceptor)
@CacheTTL(60)
@Get('products')
findAll() { ... }
// 캐시 키: tenant:{id}:GET:/products
```

- 기존 `@nestjs/cache-manager`와 호환
- `AsyncLocalStorage`에서 tenant ID를 읽어 자동 prefix
- `sharedCache: true` 옵션으로 테넌트 무관 캐시 지원
- **난이도:** 중 | **임팩트:** 높 | **의존성:** `@nestjs/cache-manager` (optional peer) | **예상 규모:** ~5 파일

### 3. WebSocket 테넌트 격리

**수요:** GraphQL Subscriptions, Socket.IO 기반 실시간 기능에서 테넌트 격리가 누락되는 사례 다수. 현재 `TenantMiddleware`는 HTTP only.

```typescript
app.useWebSocketAdapter(
  new TenantAwareWsAdapter(app, {
    extractor: new HeaderTenantExtractor('X-Tenant-Id'),
  }),
);
```

- `socket.io` / `ws` 핸드셰이크에서 tenant 추출
- `WsGuard`에 `@BypassTenancy` 지원 확장
- 기존 `TenancyGuard`의 `executionContext.getType() !== 'http'` skip 로직을 `'ws'` 지원으로 확장
- **난이도:** 중-고 | **임팩트:** 높 | **의존성:** 없음 (structural typing) | **예상 규모:** ~6 파일

## Tier 2 — 중요 (v0.11.0 목표, 진행 상황에 따라 결정)

### 4. Per-Tenant Rate Limiting

**수요:** noisy neighbor 방지. `@nestjs/throttler`는 글로벌/IP 기반만 지원하고 테넌트별 제한 없음.

```typescript
@TenantThrottle({ limit: 100, ttl: 60 })
@Get('api/data')
getData() { ... }
// tenant별 독립 버킷: tenant:abc → 100req/60s
```

- `@nestjs/throttler` 확장 guard
- 테넌트별 독립 버킷 (Redis 또는 in-memory)
- 한도 초과 시 `tenant.rate_limited` 이벤트 emit
- **난이도:** 중 | **임팩트:** 중 | **예상 규모:** ~4 파일

### 5. Tenant Provisioning Service

**수요:** 새 테넌트 온보딩 자동화. 현재 CLI는 스키마 전체에 대해 SQL을 생성하지만, 런타임에 새 테넌트를 프로비저닝하는 API가 없음.

```typescript
@Injectable()
export class OnboardingService {
  constructor(private readonly provisioning: TenantProvisioningService) {}

  async createTenant(tenantId: string) {
    await this.provisioning.provision(tenantId, {
      seedData: async (prisma) => {
        await prisma.setting.create({ data: { key: 'plan', value: 'free' } });
      },
    });
  }
}
```

- `beforeProvision` / `afterProvision` 훅
- 초기 데이터 시딩 콜백
- 테넌트 상태 관리 (`active`, `suspended`, `archived`)
- suspended 테넌트 요청 자동 거부 guard
- **난이도:** 중-고 | **임팩트:** 중 | **예상 규모:** ~8 파일

## Tier 3 — 향후 (v0.12.0+)

| 기능 | 설명 | 이유 |
|------|------|------|
| Schema-per-tenant 하이브리드 | 프리미엄 테넌트에 전용 스키마, 일반은 RLS | 엔터프라이즈 수요. Prisma multi-schema GA 활용 |
| Cross-tenant admin 쿼리 | `bypassRls()` + 감사 로그 | 어드민 대시보드 필수 기능 |
| Data residency 라우팅 | 테넌트별 리전 DB 라우팅 | GDPR/규제 요건 |
| Prisma 7 호환성 | Rust 엔진 제거 후 호환 검증 | Prisma 7 릴리즈 시점에 맞춰 대응 |

| 순서 | 기능 | 타입 | 예상 규모 | Tier |

| 1 | Tenant-Scoped Logger | feat | 소 (~3 파일) | 1 |
| 2 | Tenant-Aware Cache Interceptor | feat | 중 (~5 파일) | 1 |
