## 검증이 필요한 항목

### 1. CurrentTenantDecorator — 커버리지 0%

- **파일:** `src/decorators/current-tenant.decorator.ts` (8줄, 전체 미커버)
- **현황:** 테스트 파일(`test/current-tenant.decorator.spec.ts`)은 존재하지만 `TenancyContext`만 테스트하고, 실제 데코레이터의 param 바인딩은 검증하지 않음
- **위험:** NestJS HTTP 요청 컨텍스트에서 tenant ID를 올바르게 추출하는지 확인되지 않음
- **보완:** NestJS `ExecutionContext` mock을 사용한 param 추출 테스트 추가

### 2. Extractor 엣지케이스

- **PathExtractor** (`path.extractor.ts:26`) — URL 인코딩된 tenant ID (`%2F`, `%3A` 등) 처리 미검증 (95%)
- **SubdomainExtractor** (`subdomain.extractor.ts:18, 47`) — 도메인 유효성 검사 엣지케이스 (90.9%)
- **전체 Extractor** — 특수문자, 빈 문자열, 초대형 문자열 등 경계값 테스트 부족
- **보완:** URL 인코딩, 특수문자, malformed input에 대한 테스트 추가

### 3. RLS 우회 시나리오 테스트

- **현황:** E2E 테스트가 격리(isolation)는 검증하지만, 정책 우회 시도(policy circumvention)는 테스트하지 않음
- **위험:** 악의적 입력이나 잘못된 정책 설정 시 데이터 노출 가능성
- **보완:** permission denied 시나리오, 잘못된 정책 구성, 우회 시도 테스트 추가

### 4. CLI init.ts 누락 경로 (83.33%)

- **미커버 라인:** 40-41 (multi-schema 모델 필터링), 100 (dry-run), 120-125 (파일 쓰기)
- **위험:** multi-schema 환경에서 잘못된 SQL 생성 가능
- **보완:** dry-run 모드, 파일 I/O 에러, prompts 패키지 누락 시 동작 테스트
